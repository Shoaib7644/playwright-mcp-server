Phase 1 Set up your Mac + IntelliJ IDEA
1
Install prerequisites
JDK 17 + Maven + Git

Open Terminal and install via Homebrew (recommended on macOS):

brew install openjdk@17 maven git
echo 'export JAVA_HOME=/opt/homebrew/opt/openjdk@17' >> ~/.zshrc
echo 'export PATH="$JAVA_HOME/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc

Verify everything works:

java -version   # should say: openjdk 17...
mvn -version    # should say: Apache Maven 3.x
git --version
On Apple Silicon Macs, Homebrew installs to /opt/homebrew — not /usr/local. The export above handles this automatically.
2
Download IntelliJ IDEA
Community (free) or Ultimate

Download from jetbrains.com/idea. The free Community Edition is sufficient — it handles Maven, Java, and run configurations fully.

During install, when IDEA asks about the JDK, point it at /opt/homebrew/opt/openjdk@17 using Add JDK → Detect or by pasting the path manually.

In IDEA: Settings → Build → Build Tools → Maven — set Maven home to the output of mvn -version (usually /opt/homebrew/opt/maven). This ensures IDEA uses the same Maven as your terminal.
Phase 2 Create the Maven project in IntelliJ
3
Create new project
Maven archetype
  • Click File → New → Project
  • In the left panel select New Project
  • Set Language to Java
  • Set Build system to Maven
  • Set JDK to 17
  • Set Group ID to com.qa.mcp
  • Set Artifact ID to playwright-mcp-server
  • Set Version to 1.0.0-SNAPSHOT
  • Name the project folder playwright-mcp-server
  • Click Create
Do NOT check "Add sample code". You will paste the provided files manually into the right locations.
4
Build the correct folder structure
Standard Maven layout

In the Project panel on the left, your final structure should look exactly like this:

playwright-mcp-server/
├── pom.xml
└── src/
    └── main/
        ├── java/
        │   └── com/
        │       └── qa/
        │           └── mcp/
        │               ├── PlaywrightMcpServer.java
        │               └── BddCodeGenerator.java
        └── resources/
            └── logback.xml

To create missing folders: right-click src/main/java → New → Package → type com.qa.mcp. IDEA creates all the nested folders in one step.

For resources: right-click src/main → New → Directory → select resources from the suggested list (IDEA will auto-mark it as a Resources Root).

5
Paste the source files
3 files to create
  • Right-click com.qa.mcp package → New → Java Class → name it PlaywrightMcpServer. Paste the full contents of that file.
  • Repeat for BddCodeGenerator.
  • Right-click the resources folder → New → File → name it logback.xml. Paste the logback config contents.
  • Finally, open the pom.xml that IDEA created at the project root and replace its entire contents with the provided pom.xml.
After pasting pom.xml, a blue banner will appear at the top of the editor saying "Maven build scripts found". Click Load Maven Project — IDEA will now download all dependencies automatically.
Phase 3 Install Playwright browsers + first build
6
Download Chromium (one-time)
Required before running

Open the Terminal tab at the bottom of IDEA (or use your Mac Terminal, cd into the project folder) and run:

mvn exec:java@install-browsers

This runs the Playwright CLI and downloads Chromium (~170 MB) to your home folder at ~/.cache/ms-playwright. It only needs to run once per machine.

If the download stalls, add -Dexec.args="install chromium --with-deps" to also install the system dependencies Playwright needs on some macOS configurations.
7
Build the fat jar
Creates the deployable artifact

Either in the IDEA Terminal or via the Maven panel (right sidebar → Lifecycle → package), run:

mvn clean package -DskipTests

This produces:

target/playwright-mcp-server-1.0.0-SNAPSHOT.jar   ← the fat jar (~60 MB)

Verify it works by doing a quick smoke-test in terminal:

java -jar target/playwright-mcp-server-1.0.0-SNAPSHOT.jar

You should see PlaywrightMcpServer running on stdio… in stderr. It will wait for JSON-RPC input — press Ctrl+C to exit.

In IDEA you can also create a Run Configuration: Run → Edit Configurations → + → Application, set Main class to com.qa.mcp.PlaywrightMcpServer. Use this for debugging with breakpoints.
8
Set up a Run Configuration for debugging
Optional but highly recommended
  • Go to Run → Edit Configurations → click +
  • Choose Application
  • Name: MCP Server (debug)
  • Module: playwright-mcp-server
  • Main class: com.qa.mcp.PlaywrightMcpServer
  • Working directory: $MODULE_WORKING_DIR$
  • Click OK

Now you can set breakpoints in handleStartRecording, handleGenerateBdd, etc. and use the IDEA debugger to step through the MCP tool logic live.

Phase 4 Integrate with GitHub Copilot agent (MCP)
9
Integrate with IntelliJ IDEA + Copilot
Via .github/copilot-mcp.json

GitHub Copilot's agent mode reads MCP server configs from a .github/copilot-mcp.json file in your project root (or from VS Code / IDEA global settings).

Create the file .github/copilot-mcp.json inside your project (or any project you want to use the server from):

{
  "servers": {
    "playwright-mcp": {
      "type": "stdio",
      "command": "java",
      "args": [
        "-jar",
        "/Users/YOUR_USERNAME/projects/playwright-mcp-server/target/playwright-mcp-server-1.0.0-SNAPSHOT.jar"
      ]
    }
  }
}

Replace /Users/YOUR_USERNAME/projects/... with the actual absolute path to your jar. You can get it by running pwd inside the project folder in Terminal.

In IDEA: open the GitHub Copilot chat panel → click the agent icon (✦) → your tool names (start_recording, stop_recording, etc.) should appear in the tool list. If not, click Refresh MCP servers.
Use absolute paths only — never relative paths like ./target/.... The MCP host process runs from a different working directory and relative paths will fail silently.
10
Integrate with Eclipse + Copilot
Via VS Code-style settings.json

GitHub Copilot in Eclipse uses the same MCP config as VS Code. Edit the global Copilot settings file at:

~/.config/github-copilot/settings.json   (macOS/Linux)

Add the following block (merge with existing content if the file already exists):

{
  "github.copilot.advanced": {},
  "mcp": {
    "servers": {
      "playwright-mcp": {
        "type": "stdio",
        "command": "java",
        "args": [
          "-jar",
          "/Users/YOUR_USERNAME/projects/playwright-mcp-server/target/playwright-mcp-server-1.0.0-SNAPSHOT.jar"
        ]
      }
    }
  }
}

Restart Eclipse. In the Copilot Chat panel, switch to Agent mode. Your Playwright tools will be available.

Eclipse's Copilot plugin version must be 1.5.0+ for MCP agent support. Check via Help → About → Installation Details and search "Copilot".
11
Test the integration end-to-end
First live recording

In the Copilot agent chat panel (IDEA or Eclipse), type:

Use start_recording with url "https://example.com"

A Chromium window will open. Interact with the page (click links, fill forms). Then type:

Use stop_recording

You will get back a JSON array of every captured interaction. Then to generate the full BDD project:

Use generate_playwright_bdd with the recording above and featureName "My First Test"

The agent will return the Feature file, Step Definitions, Page Object, and pom.xml all inline in the chat.

Copy each generated file into the correct folder of a new Maven project. Run mvn test to execute the BDD scenario against a live browser.
Phase 5 Build, publish and share the server
12
Version and tag a release
Git + Maven versions plugin

When ready to share a stable version, bump the version and tag it:

mvn versions:set -DnewVersion=1.0.0
git add pom.xml
git commit -m "chore: release v1.0.0"
git tag v1.0.0
git push origin main --tags
Use mvn versions:revert if you set the wrong version — it restores the backup pom.xml that the plugin automatically creates.
13
Publish to GitHub Packages
Share within your org

Add this to your pom.xml inside the <project> block:

<distributionManagement>
  <repository>
    <id>github</id>
    <url>https://maven.pkg.github.com/YOUR_ORG/playwright-mcp-server</url>
  </repository>
</distributionManagement>

Add a token to ~/.m2/settings.xml:

<servers>
  <server>
    <id>github</id>
    <username>YOUR_GITHUB_USERNAME</username>
    <password>ghp_YOUR_PAT_TOKEN</password>
  </server>
</servers>

Then deploy:

mvn clean deploy -DskipTests
The PAT token needs write:packages scope. Create it at github.com → Settings → Developer settings → Personal access tokens.
14
Share the fat jar directly (simpler)
No registry needed

For team distribution, the fat jar is self-contained — teammates only need Java 17 installed. Upload target/playwright-mcp-server-1.0.0-SNAPSHOT.jar to:

  • A GitHub Release asset (attach via the Releases UI)
  • Your team's shared network drive
  • An S3 bucket with a download link

Teammates then just update the args path in their copilot-mcp.json to point to wherever they placed the jar locally.

Document that teammates must also run the one-time browser install: java -cp playwright-mcp-server-1.0.0.jar com.microsoft.playwright.CLI install chromium
